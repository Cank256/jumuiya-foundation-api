> `NEXT_PUBLIC_API_URL` (your Laravel app) and returns the response unchanged.

## 1. Architecture Overview

```
Browser → Next.js (/api/*) → Laravel ({NEXT_PUBLIC_API_URL}/*)
```


The environment variable `NEXT_PUBLIC_API_URL` in `.env.local` points to your Laravel base URL,
e.g. `https://api.jumuiyafoundation.org`. The Next.js route files that already exist are:

| Next.js Route File | Proxies to Laravel |
|---|---|
| `GET /api/events` | `GET {API_URL}/events` |
| `GET /api/events/:id` | `GET {API_URL}/events/:id` |
| `GET /api/news` | `GET {API_URL}/news` |
| `GET /api/news/:id` | `GET {API_URL}/news/:id` |

The remaining endpoints listed in this document are called directly by page components
but have **no proxy route file yet** — those files also need to be created in Next.js
(see [Section 13](#13-nextjs-proxy-routes-still-needed)).

## 2. General Conventions

### Request Headers
Every Next.js proxy sends:
```
Accept: application/json
```
Your Laravel API should always respond with:
```
Content-Type: application/json
```

### Response Envelope
List endpoints must wrap results in a `data` key:
```json
{
  "data": [ ... ]
}
```
Single-resource (show) endpoints return a **plain object with no envelope**:
```json
{ "id": 1, "title": "..." }
```


### Date Format
All dates must be **ISO 8601 strings**. The frontend calls `new Date(dateStr)` directly.
```
"2025-09-15T09:00:00Z"   // preferred — UTC
"2025-09-15"             // acceptable for date-only fields
```

### HTML Content Fields
Many text fields (`description`, `content`, `responsibilities`, `requirements`, etc.) may
contain rich HTML from an admin editor. The frontend detects HTML via the regex
`/<[a-z][\s\S]*>/i` and renders it with `dangerouslySetInnerHTML`; otherwise plain text is
rendered. Store and return whichever format the admin enters.

### Image URLs
All image URL fields (`featured_image`, `download_url`, etc.) must be **absolute URLs**, e.g.
`https://api.jumuiyafoundation.org/storage/images/article.jpg`. Next.js `<Image>` uses them
directly. Remember to add your Laravel domain to `remotePatterns` in `next.config.ts`.

### HTTP Status Codes
| Scenario | Status |
|---|---|
| Success (list or show) | `200 OK` |
| Resource not found | `404 Not Found` |
| Validation error | `422 Unprocessable Entity` |
| Server error | `500 Internal Server Error` |

Error body:
```json
{ "error": "Human-readable message" }
```
Or Laravel's default validation shape:
```json
{ "message": "...", "errors": { "field": ["..."] } }
```

## 3. Events

### `GET /events`

Returns all events. The frontend splits them into "upcoming" and "past" sections
client-side — return **all** events and let the frontend filter.

**Response** `200 OK`
```json
{
  "data": [
    {
      "id": 1,
      "title": "Annual Community Forum 2025",
      "description": "<p>HTML or plain text description.</p>",
      "category": "Forum",
      "location": "Kampala, Uganda",
      "start_date": "2025-09-20T09:00:00Z",
      "end_date": "2025-09-21T17:00:00Z",
      "time": "9:00 AM – 5:00 PM EAT",
      "seats": "Limited seats available",
      "status": "upcoming",
      "featured": true,
      "featured_image": "https://api.example.org/storage/events/forum.jpg",
      "registration_url": "https://forms.example.org/register"
    }
  ]
}
```


**Event Object Fields**

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | integer | ✅ | Primary key |
| `title` | string | ✅ | |
| `description` | string\|null | | HTML or plain text |
| `category` | string\|null | | `Forum`, `Workshop`, `Training`, `Fundraising`, `Symposium`, `Networking`, `Conference` |
| `location` | string\|null | | |
| `start_date` | ISO datetime\|null | | |
| `end_date` | ISO datetime\|null | | |
| `time` | string\|null | | Human-readable, e.g. `"9:00 AM – 5:00 PM"` |
| `seats` | string\|null | | Human-readable, e.g. `"Limited seats"` |
| `status` | string | | `upcoming`, `ongoing`, `completed` |
| `featured` | boolean\|null | | Pins event to hero position on list page |
| `featured_image` | string\|null | | Absolute URL |
| `registration_url` | string\|null | | External registration link |

**Frontend filtering logic:**
- Upcoming: `status === 'upcoming'` OR `status === 'ongoing'` OR `start_date >= now`
- Past: `status === 'completed'` OR `(start_date < now AND status not upcoming/ongoing)`
- Featured: `featured === true` → rendered as the hero card

---

### `GET /events/:id`

**Response** `200 OK` — same fields as above, **no envelope**, single object.

**Response** `404 Not Found`
```json
{ "error": "Event not found" }
```

### `GET /news/:id`

The `:id` segment may be either a **numeric ID** or a **slug** — the frontend uses
`slug ?? id` when building links. Your route should resolve both:
```

```

**Response** `200 OK` — single `NewsArticle` object (no envelope), includes `content` and `creator`.

**Response** `404 Not Found`
```json
{ "error": "Article not found" }
```

### `GET /tenders`

**Response** `200 OK`
```json
{
  "data": [
    {
      "id": 1,
      "title": "Supply of IT Equipment for District Offices",
      "reference_number": "JDF/PROC/2025/004",
      "description": "Brief description of procurement scope.",
      "deadline": "2025-08-15T17:00:00Z",
      "created_at": "2025-07-01T09:00:00Z",
      "status": "open"
    }
  ]
}
```

Note: frontend filters client-side to `status === 'open'`.


**Tender Object Fields**

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | integer | ✅ | |
| `title` | string | ✅ | |
| `status` | string | ✅ | `open`, `closed`, `awarded` |
| `reference_number` | string\|null | | e.g. `"JDF/PROC/2025/004"` — displayed in monospace |
| `description` | string\|null | | HTML or plain text |
| `requirements` | string\|null | | HTML or plain text — detail page only |
| `deadline` | ISO datetime\|null | | Frontend shows "Closing soon" if ≤ 7 days away |
| `created_at` | ISO datetime\|null | | Published date shown on detail page |
| `document_url` | string\|null | | Fallback single document URL |
| `has_rfp_document` | boolean\|null | | Whether an RFP file is attached |
| `rfp_download_url` | string\|null | | Absolute URL |
| `rfp_document_name` | string\|null | | Filename label |
| `rfp_document_size` | integer\|null | | Bytes |
| `has_tor_document` | boolean\|null | | Whether a Terms of Reference file is attached |
| `tor_download_url` | string\|null | | Absolute URL |
| `tor_document_name` | string\|null | | Filename label |
| `tor_document_size` | integer\|null | | Bytes |
| `documents` | TenderDocument[]\|null | | **Preferred** structured document array (see below); takes priority over flat fields |

**TenderDocument sub-object:**
```json
{
  "id": 1,
  "name": "JDF-RFP-2025-004.pdf",
  "type": "rfp",
  "url": "https://api.example.org/storage/tenders/rfp.pdf",
  "size": 1048576
}
```

| Field | Type | Notes |
|---|---|---|
| `id` | integer\|null | |
| `name` | string\|null | Display filename |
| `type` | string | `rfp`, `tor`, `specification`, `other` |
| `url` | string | Absolute download URL |
| `size` | integer\|null | Bytes — displayed as `"1.0 MB"` |

**Document resolution priority (frontend logic):**
1. If `documents` array is present and non-empty → use it
2. Else if `has_rfp_document && rfp_download_url` → build RFP entry from flat fields
3. Else if `has_tor_document && tor_download_url` → build ToR entry from flat fields
4. Else if `document_url` → use as generic fallback

## 11. Analytics — Frontend to Backend

The frontend collects three types of analytics events and **posts them directly to the
Laravel backend** over `NEXT_PUBLIC_API_URL`. There is no Next.js proxy layer for these —
they are sent straight from the browser. Laravel stores them for display in the admin panel.

All requests include:
```
Content-Type: application/json
```
All calls are fire-and-forget (`keepalive: true`) and failures are silently swallowed on the
frontend, so these endpoints must never return a client-breaking response.

---

### `POST /analytics/event`

Fired automatically on every page navigation (via `AnalyticsProvider`) and on key user
interactions (button clicks, form submissions).

**Request body variants:**

```json
// Page view — fired on every route change
{
  "type": "page_view",
  "path": "/news/community-outreach-june-2025",
  "title": "Community Outreach Programme Reaches 500 Families | Jumuiya Development Foundation",
  "timestamp": "2025-07-01T10:23:45.000Z"
}
```

```json
// Button click
{
  "type": "button_click",
  "buttonName": "Donate Now",
  "section": "hero",
  "timestamp": "2025-07-01T10:24:10.000Z"
}
```

```json
// Form submission
{
  "type": "form_submission",
  "formName": "Application Form",
  "success": true,
  "timestamp": "2025-07-01T10:25:00.000Z"
}
```

**Field reference**

| Field | Type | Always present | Notes |
|---|---|---|---|
| `type` | string | ✅ | `page_view`, `button_click`, `form_submission` |
| `timestamp` | ISO datetime string | ✅ | Browser UTC time |
| `path` | string | page_view only | URL path + query string |
| `title` | string | page_view only | `document.title` at navigation time |
| `buttonName` | string | button_click only | |
| `section` | string | button_click only | Page section identifier |
| `formName` | string | form_submission only | |
| `success` | boolean | form_submission only | Whether submission succeeded |

Laravel should also capture the following from the request headers / connection:

| Extra field | Source |
|---|---|
| `ip` | `X-Forwarded-For` header or connection IP |
| `user_agent` | `User-Agent` header |
| `referer` | `Referer` header |
| `session_id` | Optional: generate a UUID per-session using a cookie |

**Response** `200 OK`
```json
{ "success": true }
```

---

### `POST /analytics/error`

Fired when the frontend catches an unexpected JavaScript error worth logging.

**Request body:**
```json
{
  "message": "Failed to fetch events",
  "context": {
    "page": "/events",
    "component": "EventsPage",
    "status": 500
  },
  "timestamp": "2025-07-01T10:26:00.000Z"
}
```

| Field | Type | Required | Notes |
|---|---|---|---|
| `message` | string | ✅ | Error message |
| `context` | object | ✅ | Arbitrary key-value diagnostic data |
| `timestamp` | ISO datetime string | ✅ | |

**Response** `200 OK`
```json
{ "success": true }
```

---

### Suggested Laravel Database Schema

```sql
-- analytics_events table
CREATE TABLE analytics_events (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  type          VARCHAR(50)  NOT NULL,          -- page_view | button_click | form_submission
  path          VARCHAR(500) NULL,
  title         TEXT         NULL,
  button_name   VARCHAR(255) NULL,
  section       VARCHAR(255) NULL,
  form_name     VARCHAR(255) NULL,
  success       TINYINT(1)   NULL,
  ip            VARCHAR(45)  NULL,
  user_agent    TEXT         NULL,
  referer       TEXT         NULL,
  session_id    VARCHAR(100) NULL,
  occurred_at   DATETIME     NOT NULL,          -- from request timestamp field
  created_at    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);

-- analytics_errors table
CREATE TABLE analytics_errors (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  message       TEXT         NOT NULL,
  context       JSON         NULL,
  ip            VARCHAR(45)  NULL,
  user_agent    TEXT         NULL,
  occurred_at   DATETIME     NOT NULL,
  created_at    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);
```

> These endpoints are public (no auth). Rate-limit them in Laravel to prevent abuse —
> e.g. 120 requests per minute per IP using `throttle:120,1`.

## 12. Next.js Proxy Routes Still Needed

The following Next.js `app/api/*/route.ts` files need to be created. Each should follow the
exact same pattern as the existing `app/api/events/route.ts`:

```
app/api/jobs/route.ts
app/api/jobs/[id]/route.ts
app/api/tenders/route.ts
app/api/tenders/[id]/route.ts
app/api/annual-reports/route.ts
app/api/contact/route.ts              (POST — forwards to Laravel /contact)
app/api/partnership-enquiry/route.ts  (POST — forwards to Laravel /partnership-enquiry)
app/api/newsletter/subscribe/route.ts (POST — forwards to Laravel /newsletter/subscribe)
```

Each GET proxy follows this template (replace `jobs` with the resource name):

```typescript
// app/api/jobs/route.ts
import { NextResponse } from 'next/server';

const API_URL = process.env.NEXT_PUBLIC_API_URL;

export async function GET() {
  if (!API_URL) {
    return NextResponse.json({ error: 'API URL not configured' }, { status: 503 });
  }
  try {
    const res = await fetch(`${API_URL}/jobs`, {
      next: { revalidate: 60 },
      headers: { Accept: 'application/json' },
    });
    if (!res.ok) {
      return NextResponse.json({ error: 'Failed to fetch jobs' }, { status: res.status });
    }
    return NextResponse.json(await res.json());
  } catch {
    return NextResponse.json({ error: 'Internal server error' }, { status: 500 });
  }
}
```
